## 概要

半構造化N:Mスパース性と低ビット量子化（例：1.58ビットBitNet）は、大規模言語モデル（LLM）の効率を向上させるための2つの有望なアプローチであるが、これまで主に独立して研究されてきた。本研究では、両者の相互作用を調査し、1.58ビットBitNetがフル精度モデルよりもN:Mスパース性と自然に高い親和性を持つことを示す。この効果を研究するために、我々はSparse-BitNetを提案する。これは1.58ビット量子化と動的N:Mスパース化を統合的に適用しつつ、初めて安定した学習を保証する統一フレームワークである。複数のモデルスケールおよび学習レジーム（スパース事前学習およびデンス-to-スパーススケジュール）にわたり、1.58ビットBitNetは同一のスパース性レベルにおいてフル精度ベースラインよりも一貫して小さな性能劣化を示し、精度崩壊が起こるまでにより高い構造化スパース性を許容できることが確認された。さらに、我々のカスタムスパーステンソルコアを使用することで、Sparse-BitNetは学習と推論の両方で最大1.30倍の大幅な高速化を達成した。これらの結果は、極低ビット量子化と半構造化N:Mスパース性の組み合わせが効率的なLLMに向けた有望な方向性であることを示している。コードは https://github.com/AAzdi/Sparse-BitNet で公開されている。

## 1 はじめに

大規模言語モデル（LLM）は幅広いタスクにおいて顕著な性能を示してきた [Ope23, Ope25, LFX⁺24, YLY⁺25]。しかし、そのスケールの急速な増大は学習と推論に多大なコストをもたらし [LFX⁺24, YLY⁺25]、効率性が現代のLLM研究における中心的な課題となっている。既存のアプローチの中で、量子化 [LTT⁺24, FAHA22, DLBZ22] とスパース性 [JCM⁺21, tor24, HZH⁺24] は、LLMの効率向上に向けた2つの広く研究された効果的な戦略として台頭してきた。

半構造化N:Mスパース性、特に広くサポートされている2:4パターン [FYM⁺24, HZH⁺24, tor24] は、NVIDIA Sparse Tensor Core [NVI22]（スパース性をハードウェアアクセラレーションに活用するために、4つの重みのうち最大2つが非ゼロであることを要求する）上で行列乗算を高速化する能力により、注目が高まっている。しかしながら、既存の研究 [FYM⁺24, HCC⁺25, KWS⁺25] は主にフル精度LLMに半構造化スパース性を適用してきた。厳格なN:M制約の下では、これらのフル精度モデルはしばしば急速な精度劣化に悩まされ、高いスパース性と高い性能の両立が困難となっている。

一方で、極低ビット量子化がLLM効率化に向けた代替的な経路として台頭してきた。特に1.58ビットBitNet [WMD⁺23, WMW25, MWH⁺25a] は重みを三値集合{−1, 0, 1}に量子化し、スケールにおいてフル精度ベースラインと競合する性能を達成する [MWH⁺25a]。図1に示すように、事前学習された1.58ビットBitNetの重みは特徴的な量子化バレー構造を示し、ゼロ値の三値重みの割合が高く（約42%）、明示的な枝刈りなしに自然にスパースな表現を形成する。これらのゼロは非構造化であり、N:Mスパースカーネルアクセラレーションを直接可能にするものではないが、マグニチュードベースのN:Mスパース性選択と本質的に高い親和性を持つ重みマグニチュードの幾何学的構造を明らかにしている。

この観察は、文献における重要でありながら未開拓の空白を浮き彫りにする。先行研究はフル精度モデルにおけるN:Mスパース性と低ビット量子化を独立に広く研究してきたが、極低ビット量子化と半構造化スパース性の相互作用はほとんど探索されていない。この空白が以下の研究課題を動機付ける：

> **同一のN:Mスパース性制約の下で、1.58ビットBitNetはフル精度モデルよりもスパース性に親和的か？**

この問いに答えるため、我々はまず統一的なSparse-BitNet学習フレームワーク（§2）を開発し、1.58ビットBitNet重み量子化とN:M半構造化スパース性制約を同時に適用しつつ、LLMの安定的な学習を保証する。

このフレームワークを用いて、1.58ビットBitNetとBF16モデルのスパース性親和性を2つの設定で比較する：(1) N:Mスパース性制約を課したスクラッチからの事前学習、および(2) 異なる段階でデンスからN:Mスパース学習に切り替えるデンス-to-スパース学習スケジュール。Qwen-2.5モデルファミリー [Tea24] を複数のスケール（0.5B〜3B）で実験した結果、1.58ビットBitNetは同一のスパース性制約の下でBF16よりも一貫して小さな性能劣化を示すことが確認された（図2参照）。

さらに、様々なシーケンス長とバッチサイズにおいて、我々の自社開発6:8スパースオペレータを使用してNVIDIA GPU上でエンドツーエンドの推論スループットを測定し、1.58ビットBitNetとN:Mスパース性の組み合わせが学習と推論の両方で大幅な高速化（最大1.30倍）をもたらすことを確認した。我々の主な貢献は以下の通りである：

- 我々は初めて体系的に調査し、極低ビット量子化（例：1.58ビットBitNet）がフル精度（例：BF16）モデルよりも半構造化N:Mスパース性と本質的に高い親和性を持ち、同一のスパース性制約下でより小さな精度劣化を示すことを明らかにした。
- 我々は1.58ビット量子化とN:Mスパース性を統合的に組み合わせ、学習の安定性と頑健性を向上させるSparse-BitNet学習フレームワークを提案した（§2）。
- 広範な実験により、Sparse-BitNetがスパースフル精度ベースラインよりも優れた精度-効率のトレードオフを達成し（図2参照）、好ましいエンドツーエンドの高速化を実現することを実証した（§3）。

## 2 Sparse-BitNet

本節では、三値量子化学習の枠組みに半構造化スパース性を統合する手法であるSparse-BitNetを提示する。まず基盤となるコンポーネント——BitNet b1.58とN:Mスパース性——を概観し、その後、提案するアーキテクチャ、学習戦略、およびこれら2つの圧縮次元間の相互作用について詳述する。

### 2.1 予備知識

**1.58ビットBitNet.** 1.58ビットBitNet [MWH⁺25b] はBitNetアーキテクチャ [WMD⁺23] の発展形であり、重みを三値集合 T = {−1, 0, +1} に制約し、理論的な情報密度は log₂3 ≈ 1.58ビット/パラメータとなる。基本的な構成要素はBitLinear層である。重み行列 W ∈ ℝ^{d_out × d_in} に対して、三値重み W_q は潜在的なフル精度重みをその平均絶対マグニチュードでスケーリングすることにより導出される：

$$W_q = \text{RoundClip}\left(\frac{W}{\gamma + \epsilon}, -1, +1\right)$$

ここで γ = (1/(d_out · d_in)) · ‖W‖₁ がスケーリング係数として機能し、εは数値安定性のための小さな定数である。活性化はabsmaxスケーリングを用いて8ビット整数に量子化される：x̃ = Quant(x) = Clip(x · 127/(max(|x|) + ε), −128, 127)。フォワードパスは y ≈ γ · matmul(W_q, x̃) として計算され、浮動小数点乗算を整数加算とスケーリングに置き換えることで計算オーバーヘッドを大幅に削減する。

**半構造化N:Mスパース性.** 半構造化スパース性は、M個の連続する重みのうち最大N個が非ゼロであるという細粒度パターンを強制する [ZMZ⁺21]。この形式は粗い枝刈りに対する柔軟性を提供しつつ、ハードウェア効率（例：Sparse Tensor Core経由）を維持する。重み行列Wに対して、バイナリマスク M ∈ {0, 1}^{d_out × d_in} を定義し、サイズMの各グループ（通常は入力次元に沿って）について ‖M_group‖₀ = N とする。スパース計算は y = (W ⊙ M)x として実行される。本研究では、低ビットLLMにおける圧縮と精度のバランスの取れたトレードオフとして6:8パターン（25%スパース性）に焦点を当て、標準的な2:4パターンとのベンチマークも行う。

### 2.2 Sparse-BitLinearアーキテクチャ

Sparse-BitNetは標準的な線形射影をSparse-BitLinear層に置き換える。この層は三値量子化とN:Mマスキングを単一のオペレータに合成し、スクラッチから学習される。最適化中は勾配を蓄積するために高精度のマスター重み W（例：bf16）を保持する。

**マグニチュードベースのマスク生成.** スパース性パターンを決定するために、マスク M_{N:M} をマスター重み W から直接計算する。マグニチュード枝刈りを採用し、サイズ M の各連続グループについて、絶対値が最大の N 個のインデックスを選択する。

$$M_{N:M} = \Pi_{N:M}(|W|)$$

ここで Π_{N:M}(·) はグループごとのTop-N指示関数である。重要な点として、マスキングは量子化前の重みに基づいて実行され、細粒度のマグニチュードランキングを保持する。これにより、離散的な三値重みにマスキングを適用した場合に生じるタイブレーキング問題を回避する。

**量子化-マスク計算.** フォワードパスは「量子化-マスク」パラダイムに従う。まず活性化 x を x̃（通常8ビット）に、マスター重み W を三値重み W_q = Q_t(W) ∈ {−1, 0, +1} に量子化する。次にマスクを量子化済み重みに適用する：

$$W_{\text{eff}} = W_q \odot M_{N:M}$$

出力はこれらの有効なスパース量子化重みを用いて計算される：

$$y \approx s \cdot (W_{\text{eff}} \tilde{x})$$

ここで s は量子化スケールを吸収する。この順序により、N:Mマスクパターンが推論に使用される最終的な離散重みに適用されることが保証され、ハードウェアカーネルのための明確に定義されたN:Mメタデータ/レイアウトが得られる。詳細なtorchスタイルの疑似コードをAlgorithm 2に提供する。

### 2.3 学習戦略

Sparse-BitNetをスクラッチから学習し、マスター重み W をエンドツーエンドで最適化する。学習手順はAlgorithm 1にまとめられている。

**動的マスク再計算.** ポストトレーニング枝刈り手法とは異なり、式(2)を用いて各学習ステップでM_{N:M}を再計算する。これは離散制約が継続的に再評価される射影最適化アプローチに対応し、マスクの陳腐化を防ぎ、ネットワークのトポロジーが重み値と共に進化することを可能にする。

**デュアルSTEによる勾配推定.** 量子化関数 Q_t(·) とマスク選択 Π_{N:M}(·) はいずれも微分不可能であるため、マスター重み W への勾配伝播のために近似が必要となる。我々はデュアルストレートスルー推定器（STE）アプローチを採用する。第一に、三値量子化器 Q_t に対しては、標準的なBitNet学習に従い、クリッピング範囲内で勾配を恒等関数として通過させる。第二に、決定的に重要な点として、スパース性マスクにもSTEを適用する。具体的には、バックワードパスにおいてマスクオペレータを透過的として扱う：

$$\frac{\partial \mathcal{L}}{\partial W} = \frac{\partial \mathcal{L}}{\partial W_{\text{eff}}} \cdot \frac{\partial W_{\text{eff}}}{\partial W} \approx \frac{\partial \mathcal{L}}{\partial W_{\text{eff}}}$$

これは、フォワードパスで枝刈りされた（マスクされた）重みを含む全てのマスター重みに勾配が流れることを意味する。これは勾配をマスクでゲーティングする手法（すなわち ∂L/∂W ≈ M ⊙ ∂L/∂W_eff）[ZMZ⁺21]とは異なり、後者はアクティブセットのみに更新を制限する。デンスな勾配更新を許可することで、枝刈りされた重みが直接フィードバックを受け取り、後続のステップでTop-N集合に再参入するのに十分な大きさまで成長する可能性があり、早期の構造的崩壊を防止する。このデンスな勾配フローの必要性はアブレーション研究（§3.3参照）で実証的に検証している。

**Algorithm 1: Sparse-BitNetの学習（三値、ステップごとのマスク、デュアルSTE）**

```
入力: データ D、マスター重み W、パターン (N, M)
for step t = 1, 2, ... do
    B ← Sample(D)
    M_{N:M} ← Π_{N:M}(|W|)        // マスク計算
    x̃ ← Q_a(Norm(x))
    W_q ← Q_t(W)
    W_eff ← W_q ⊙ M_{N:M}         // マスク適用
    y ≈ s · W_eff x̃               // フォワードパス
    バックワード:
        (1) Q_t(·) を通じたSTE
        (2) Π_{N:M}(·) を通じたSTE（勾配をマスクしない）
    全マスター重み W をオプティマイザで更新
end for
```

## 3 実験

### 3.1 実験設定

**バックボーンモデル.** Qwen2.5 [Tea24] アーキテクチャに基づく3つのモデルスケールを検討する：Qwen2.5-0.5B、Qwen2.5-1.5B、Qwen2.5-3B。特に断りのない限り、全てのモデルは同一のデータ混合とトークン予算の下でスクラッチから学習される。

**スパース性パターン.** 主要なスパース学習実験では半構造化6:8スパース性を使用する。具体的には、重みはM=8要素のブロックに分割され、マグニチュードベースのルールに従って各ブロックでN=6個の重みを保持し、残りの位置をゼロに設定する。我々の手法をSparse-BitNetと呼び、この6:8制約を学習中の三値BitNetスタイルの重み量子化と組み合わせる。

**学習データと目的関数.** 全モデルサイズをRefineWeb [PMH⁺23] データで約50Bトークン学習する。学習目的関数は標準的な因果言語モデリング（次トークン予測）である。報告されるパープレキシティは全て、同一のデータ分布のホールドアウト検証分割上で計算される。

BF16とBitNetのバリアント間で、アーキテクチャ、データ混合/トークン予算、オプティマイザ、学習率スケジュールを同一に保つ。主要な違いはBitNetスタイル学習における三値重み/活性化量子化オペレータの使用である。詳細な学習設定は付録Aに記載されている。

**評価設定.** モデル品質は検証パープレキシティとダウンストリームタスク性能の両方で評価する。具体的には、5つの広く使用されるベンチマークの精度を報告する：HellaSwag [ZHB⁺19]、ARC-E [CCE⁺18]、PIQA [BZLB⁺20]、BoolQ [CLC⁺19]、COPA [RBG11]。効率の評価には、NVIDIA A100およびB200 GPU上での学習・推論スループットをベンチマークする。

### 3.2 主要結果

**ベースラインとの比較.** Sparse-BitNetを全モデルサイズにわたり以下のベースラインと比較する：(i) スパース性なしのデンスBF16事前学習、(ii) 同一の6:8スパース性制約下でのスパースBF16事前学習、(iii) スパース性なしのデンスBitNet（三値）事前学習、(iv) 6:8スパース性下でのスパースBitNet（我々の手法）事前学習。

**Sparse-BitNetはスケール間で一貫して6:8スパース化に対しより頑健である.** Qwen2.5モデルの0.5B/1.5B/3Bで6:8半構造化スパース性を評価し、各手法の自身のデンスベースラインに対する増分劣化（Δ、表1および表2）によりスパース性親和性を測定する。

**BitNetは増加する半構造化スパース性の下で遅延崩壊を示す.** 主要実験では6:8スパース学習に焦点を当てているが、Qwen2.5-0.5Bでデンス8:8から攻撃的な2:8までのN:8半構造化パターンファミリーをスイープすることにより、頑健性をさらにストレステストする。各パターンについて、対象のN:8制約下でモデルをスクラッチから学習し、検証パープレキシティ（PPL）を報告する。異なる絶対PPLスケールを持つフォーマット間の公平な比較を可能にするため、各手法のデンス対応物に対する正規化PPL増加を報告する：NormPPL(N:8) = PPL(N:8)/PPL(8:8)。

図2は、BF16がスパース性がより攻撃的になるにつれて急速に劣化することを示している。特に、ハードウェア関連の2:4設定では、BF16ベースラインは+18.8%の正規化PPL増加を被り、10%劣化閾値を超える一方、BitNetはわずか+5.7%の増加で安定を維持する。10%閾値を「崩壊」の実用的な指標として使用すると、BF16は4:8で閾値を超えるのに対し、BitNetはそれを下回り続け、3:8でようやく超える。これらの結果は、三値BitNetモデルが品質崩壊前により強力な半構造化スパース性を維持でき、デプロイメントのためのより広い実行可能なスパース性範囲を提供することを示している。

### 3.3 アブレーション研究

**動的6:8マスクの最適化設計.** Sparse-BitNetは三値重みと半構造化6:8接続性を同時に最適化する。各ステップで、現在のフル精度マスター重みからバイナリマスク m(·) を生成し、フォワードパスで6:8制約を適用する。三値量子化と動的top-N選択はいずれも微分不可能であるため、頑健な学習レシピは3つの結合した設計選択に依存する：(i) マスクされた重みが依然として勾配更新を受けられるか、(ii) マスクが連続的なマスター重みから構築されるか離散的な三値重みから構築されるか、(iii) マスキングが三値量子化の前か後に適用されるか。

Qwen2.5-0.5Bの6:8スパース性で同一の学習設定下にて以下の4つのバリアントを比較する（図3a、3b参照）：

- **ベースライン（我々の手法）：量子化-マスク + デンス勾配フロー.** マスター重み w（連続的ランキング）からマスクを計算し、三値量子化 Q_t(·) を適用し、離散重みにスパース性を適用：W_eff = m(w) ⊙ Q_t(w)。バックプロパゲーション時、全マスター重み（現在マスクされているものを含む）が勾配更新を受けるようSTEを使用。

- **勾配なしマスク（マスクされた重みに対するストップ勾配）.** ベースラインと同一のフォワード計算だが、マスクされたエントリの勾配をブロック（すなわち勾配に現在のマスクを乗算）し、マスクされた重みは更新されない。

- **量子化重みからのマスク（量子化マスク選択）.** まずマスター重みを量子化し、次に三値重みから6:8マスクを構築、すなわちtop-N選択に m(Q_t(w)) を使用。フォワードパスは W_eff = m(Q_t(w)) ⊙ Q_t(w) となる。

- **量子化前のスパース化（マスク-量子化）.** まず6:8マスクをマスター重みに適用し、次にマスクされた重みのみを量子化：W_eff = Q_t(m(w) ⊙ w)。

**パープレキシティと収束.** 図3aは、我々のベースラインが最良の収束と最低の最終パープレキシティを達成することを示す。マスクされた重みの勾配をブロックすると一貫して最適化が悪化し、より高いPPLにつながる。さらに顕著なことに、量子化された三値重みからマスクを構築すると学習が大幅に不安定化し、競争力のあるパープレキシティに到達できず、はるかに悪いレベルで停滞する。最後に、量子化前にスパース性を適用する（マスク-量子化）方法もベースラインに劣り、量子化とマスキングの合成順序が安定した最適化にとって重要であることを示している。

**フリップレートによるマスクダイナミクス.** 最適化ダイナミクスを理解するために、先行研究 [HZH⁺24] に従いマスクフリップレートを監視する。w_t をステップ t における D 次元重みベクトル、m(w_t) ∈ {0, 1}^D を対応する6:8マスクとする。フリップレートを以下のように定義する：

$$r_t = \frac{\|m(w_t) - m(w_{t-1})\|_1}{D} \in [0, 1]$$

直感的に、r_t はスパース接続パターンがどの程度頻繁に変化するかを測定する。健全なスパース学習プロセスは典型的に探索から収束への挙動を示す：初期の非自明なフリップ（マスク構成の探索）に続いて、マスクが安定するにつれて徐々に減衰する。

図3bに示すように、我々のベースラインはこの望ましい軌道に従い、初期に大幅な探索があり、後に滑らかに安定化する。対照的に、**勾配なしマスク**は学習全体を通じてはるかに低いフリップレートを示し、マスクの早期凍結を示唆する：重みがマスクされると更新されてtop-N集合に再参入できなくなり、探索が制限され最終解が劣化する。**量子化重みからのマスク**バリアントは初期に持続的に大きくノイジーなフリップレートを示し、不安定なマスク選択と一致する。この挙動は、三値量子化がマグニチュードに多くのタイ（例：0または±1の多くの重み）を作り出し、top-N選択が小さな摂動やタイブレーキングに敏感になり、安定した収束を妨げるためである。最後に、**量子化前のスパース化**はベースラインに対してフリップレートを低下させ、より悪いパープレキシティに収束し、マスキング後の量子化が量子化ノイズ/スケール推定を現在のスパースサブセットと望ましくなく結合させ、効果的な探索を妨げることを示している。

**テイクアウェイ.** これらのアブレーションは我々の最終的な学習設計を正当化する：(i) マスクされたマスター重みに勾配更新を許可し、モデルがスパース接続性を継続的に探索・修正できるようにする、(ii) タイ駆動の不安定性を避けるため三値重みではなく連続マスター重みからマスクを構築する、(iii) 推論のための明確に定義された6:8スパース離散重みを生成しつつ学習中の安定した最適化挙動を維持するため、量子化-マスクフォワードパスによりスパース性を適用する。

**デンス-to-スパース学習スケジュール.** 次に、学習軌道が半構造化スパース性下での最終品質に影響するかを研究する。具体的には、まずデンスに学習し、次に残りのステップで6:8スパース学習に切り替える2段階スケジュールを採用する。ρ ∈ {0, 25, 50, 75, 100} をスパースフェーズに費やされる総学習予算の割合（%）とし、ρ=0はデンスのみの学習、ρ=100はスクラッチからのスパース学習に対応する。

**表4: デンス-to-スパーススケジュールの検証PPLへの影響（6:8、Qwen2.5-0.5B）**

| スパース比率 ρ (%) | 検証PPL ↓ |
|-------------------|-----------|
| 25 | 27.48 |
| 50 | 27.39 |
| 75 | 26.71 |
| 100（スクラッチからスパース） | 26.31 |
| デンス学習 | 25.99 |

表4に示すように、スパース性への切り替えが遅いと収束が大幅に悪化する。特に、最後の25%または50%のステップのみをスパースに学習した場合、検証PPLはそれぞれ27.48と27.39に劣化し、スクラッチからのスパース学習の26.31と比較して悪化する。スパース予算を増やすと結果が改善する：75%のスパースステップの使用でPPLは26.71に低下するが、全体をスパースに学習する場合よりも依然として悪い。全体として、これらの結果は効果的な6:8学習には十分なスパース適応予算が必要であり、同一の総学習予算下でスパース性への切り替えを遅延させると持続的な品質ギャップが生じることを示している。

**マスター重みからのマスク vs. 量子化重みからのマスク.** Sparse-BitNetにおける第二の重要な設計選択は、6:8枝刈りマスクの構築にどの重みを使用するかである。我々のデフォルト実装ではフル精度マスター重み（BF16）からマスクを計算し、フォワードパスでマスクされた重みに三値量子化を適用する。代替案として、量子化された三値重みから直接マスクを計算する方法がある。一見自然に思えるが、三値重みはマグニチュードに多くのタイ（例：|w| ∈ {0, 1}）を導入するため、各8要素ブロック内でのtop-|w|選択が悪条件となり、潜在的に不安定になる。

Qwen2.5-0.5Bの6:8スパース学習でこれら2つのオプションを評価する。マスター重みを用いたマスク生成は検証PPL 26.31を達成するのに対し、量子化重みからのマスク生成は32.23への劇的な劣化を引き起こす。この大きなギャップは、マスク選択がマグニチュードが信頼性の高いランキング信号を提供する連続マスター重み空間で実行されるべきことを確認する。対照的に、量子化空間でのマスキングは深刻なタイ効果に悩まされ、最適化と最終品質が大幅に悪化する。

### 3.4 分析

**三値QATは単峰性収縮ではなく偏極化を誘導する.** Sparse-BitNetが6:8スパース性下で優れた頑健性を示す理由をよりよく理解するために、明示的な枝刈りに先立つ潜在重みのデンス学習ダイナミクスを分析する。近ゼロマス（P(|w|/mean(|w|) < 0.5)として定義）を追跡する。図4に示すように、BF16ベースラインはゼロ付近での持続的な集中を示し、「重要な」重みと「冗長な」重みが構造的に絡み合った単峰性分布を形成する。対照的に、BitNetは明確な偏極化傾向を示す：近ゼロマスが時間とともに大幅に減少し、潜在重みが曖昧な領域から決定的なマグニチュードに向かって能動的に移動していることを示す。これにより、「死んだ」ゾーンから分離された明確な「アクティブ」クラスターを特徴とする構造化された三峰性ヒストグラム（図5）が得られる。これは、BitNetのデンス最適化が自然にソフトセレクタとして機能し、重みをスパース性と互換性のあるトポロジーに事前分類することを示唆している。

**BitNetではブロックごとの閾値がアクティブ重みから分離される.** スパース性はブロックごとの閾値 t = |w|_(N+1) によって局所的に決定される。主要な問題は、この局所閾値が信号を「切り込む」か安全にノイズを除去するかである。図6は正規化重み |w| の分布（青）と枝刈り閾値 t の分布（橙）を重ねて示す。

BF16（左）では、閾値分布が重み分布に密接に追従する。これは強い結合を意味する：枝刈り境界が重み集団の主要部分と頻繁に交差し、重要な情報の除去が不可避となる。対照的に、BitNet（右、特に後期層）では、我々が**マグニチュード層化**と呼ぶ現象を示す。重み分布（青）はより高いマグニチュード（例：x ∈ [0.5, 1.5]）に二次的な「アクティブ」モードを発展させる。決定的に重要なことに、閾値分布（橙）は低い領域（x < 0.5）に集中し、アクティブモードに到達する前に急激に減衰する。これは構造的な分離を示す：N:M選択境界はノイズ/冗長パラメータ空間内でほぼ排他的に動作し、高マグニチュード重みの集団をほぼ完全に保持する。このメカニズムは、Sparse-BitNetが攻撃的な枝刈り制約の下でも構造的完全性を維持する理由を説明する。

## 4 関連研究

### 4.1 量子化とBitNetアーキテクチャ

量子化手法 [AZY⁺24, CCKDS23, LJK⁺23, LOZ⁺24, DZC⁺24] は、スクラッチからの学習または事前学習モデルの変換を通じて重みを圧縮（例：16ビットから8ビット）することで効率を最適化する。極端なバイナリまたは三値量子化を総称してBitNetと呼ぶ。標準的なBitNet [WMD⁺23] が重みを1ビットに制約するのに対し、BitNet b1.58 [MWM⁺24] は三値 {−1, 0, 1} スキームを採用する。ゼロを組み込むことにより、このアプローチは容量を大幅に向上させ、フル精度モデルと同等の性能を達成し [WMW24, MWH⁺25b, WMW25]、極端な量子化と標準アーキテクチャの間のギャップを効果的に橋渡しする。

### 4.2 モデル枝刈り

モデル枝刈りは非構造化アプローチと構造化アプローチに大別される。[HPTD15] によって先駆けられた非構造化枝刈りは、低マグニチュードの個々の重みを対象とする。LLM向けの最近の適応 [JCM⁺21, SLBK23, DLT⁺24, ZBL⁺24] は最小限の性能低下で高いスパース性を達成するが、直接的なハードウェアアクセラレーションを欠くことが多い。このギャップを埋めるために、N:Mスパース性 [MLP⁺21] がハードウェアフレンドリーなアクセラレーションを可能にする細粒度の構造的制約を導入する。

一方、構造化枝刈りはアテンションヘッドや行/列などの一貫したアーキテクチャコンポーネントの除去により、明示的にハードウェア効率を対象とする [MFW23, XGZC23, AZY⁺24, ACN⁺24]。これは推論高速化を保証するが、通常大幅な精度劣化を伴い、高コストな再学習を必要とする。

決定的に重要な点として、両パラダイムは一般に事前学習されたデンスモデルに対する事後的な最適化として動作する。これらのアプローチとは異なり、我々の手法は学習中に動的にスパース性を導入する。このパラダイムシフトにより、推論デプロイメントだけでなく学習効率の大幅な向上にもスパース性を活用することが可能となる。

## 5 結論

我々はSparse-BitNetを提示した。これは1.58ビット三値LLMがBF16対応物よりも半構造化スパース性に対して本質的に頑健であることを確立する統一フレームワークである。動的6:8マスキングと三値量子化を統合することにより、我々の手法は性能劣化を大幅に低減し、様々なスケールにわたってモデル崩壊を遅延させる。アブレーション研究は、デンスマスター重みからのマスク計算とマスクされた領域を通じた勾配フローの維持が効果的な最適化にとって重要であることを強調する。さらに、我々のカスタム6:8カーネルは最大1.30倍の実用的な推論高速化を実証し、極端な量子化と構造化枝刈りの組み合わせが効率的なLLMデプロイメントに向けた実行可能なパレートフロンティアを提供することを確認する。

### B Sparse-BitLinearの疑似torchスタイル実装

```python
# 明示的なForward/Backward挙動を定義するカスタムAutograd関数
class WeightQuantMasked(torch.autograd.Function):
    @staticmethod
    def forward(ctx, w, N, M):
        # 1. 動的N:Mマスキング（量子化前）
        # ランキングを保持するためデンス重みwでマスクを計算
        mask = compute_nm_mask(w, N, M)
        # 2. 三値量子化（1.58ビット）
        # スケール -> 丸め -> クランプ -> 再スケール
        scale = w.abs().mean().clamp(min=1e-5)
        w_q = (w_masked / scale).round().clamp(-1, 1) * scale
        w_masked = w_q * mask
        return w_masked

    @staticmethod
    def backward(ctx, g_out):
        # デュアルSTE実装:
        # 勾配はN:Mマスク（ゼロ）と量子化丸めの両方を無視して
        # デンス重みにストレートスルーで流れる
        # g_w = g_out * 1.0
        return g_out, None, None

class SparseBitLinear(nn.Linear):
    def __init__(self, in_features, out_features, N=2, M=4):
        super().__init__(in_features, out_features)
        self.N, self.M = N, M

    def forward(self, x):
        # 1. 活性化量子化（8ビットAbsMax）
        s_x = 127.0 / x.abs().max(dim=-1, keepdim=True).values.clamp(min=1e-5)
        x_q = (x * s_x).round().clamp(-128, 127) / s_x
        # 2. 重み量子化＆マスキング（STEあり）
        # カスタムautograd関数を適用
        w_q = WeightQuantMasked.apply(self.weight, self.N, self.M)
        # 3. 線形射影
        return F.linear(x_q, w_q, self.bias)
```
