# twilog-bookmarks

指定した日付（JST）にTwilogでブックマークしたツイートを収集し、決まったMarkdownフォーマットでファイル出力するスキル。

## 概要

`mcp__twilog__get_twitter_posts` を使ってTwilogから自分のブックマーク一覧を取得し、日本語の見出しを付けた `bookmarks-YYYY-MM-DD.md` を出力する。1日単位でも、日付範囲指定でもまとめて処理できる。

## 起動方法

ユーザーが以下のように依頼すると起動する。

- 「2026/4/8のtwilogのブックマークをまとめて」
- 「/twilog-bookmarks 2026-04-08」
- 「/twilog-bookmarks 2026/3/25 - 2026/4/8」

## 引数の形式

### 単一日付

以下のいずれかを許容する。

- `2026/4/8`
- `2026-04-08`
- `20260408`

引数が空の場合はJSTの「今日」を対象にする。

第2引数で出力先パスを指定可能（単一日付のときのみ）。指定がなければカレントディレクトリの `bookmarks-YYYY-MM-DD.md` に書く。

### 日付範囲

開始日と終了日を以下の形式で指定する。両端を含む。

- `2026/4/3 - 2026/4/8`
- `2026-04-03..2026-04-08`

範囲指定時は出力先パスの指定は受け付けず、必ず1日1ファイル（`bookmarks-YYYY-MM-DD.md`）で出力される。

## 出力フォーマット

各エントリは以下の形式で、`createdAt` 昇順（古い順）に並ぶ。

```
## {日本語の見出し1}
{createdAt(UTC ISO8601)} {authorId}
[oembed {contentUrl}]

## {日本語の見出し2}
{createdAt} {authorId}
[oembed {contentUrl}]
```

- 見出しはツイート本文をもとにした30〜60文字目安の日本語名詞句
- `createdAt` はAPIレスポンスのUTC値をそのまま使う（JSTには変換しない）
- `authorId` は `@` 無しでそのまま
- `contentUrl` は `[oembed ...]` の中にそのまま埋め込み
- エントリ間は空行1つ

## 処理フロー

### 単一日付

1. **日付正規化**: 引数を `YYYY-MM-DD`（JST）と `YYYYMMDD`（API用）に変換
2. **取得**: `mcp__twilog__get_twitter_posts` を `timeline_filter=bookmarks` で呼び、`maxPage` 分ページネーション
3. **JSTフィルタ**: 取得後にクライアント側で `createdAt`（UTC）を `+9h` してJST日付が一致するもののみ残す
4. **昇順ソート**: `createdAt` の古い順
5. **見出し生成**: 各ツイート本文（URL/ハッシュタグ/改行除去）から日本語の名詞句を作成
6. **書き出し**: 指定フォーマットで `Write`
7. **報告**: 件数と出力パスを1〜2行で報告。0件ならファイルを作らず「該当なし」

### 範囲指定

1. `START_DATE` から `END_DATE` まで1日ずつ単一日付の手順を繰り返す
2. 1日分が失敗しても他の日は続行
3. 完了後、各日の件数と出力パス（または0件スキップ）を一覧で報告

## エッジケース

| 状況 | 動作 |
|---|---|
| 日付として解釈できない引数 | 正しい形式を提示して停止（推測しない） |
| 開始日 > 終了日 | エラーで停止 |
| 範囲が32日以上 | 実行前に件数と日数を提示してユーザー確認 |
| 出力ファイルが既存 | 上書き前に件数とパスを提示してユーザー確認 |
| APIエラー / 空レスポンス | 報告してファイルを作らない |
| ページ取得の途中エラー | 何ページまで取れたか明示して停止、部分結果のファイルは作らない |

## 使用ツール

- `mcp__twilog__get_twitter_posts` — Twilogからのブックマーク取得
- `Write` — ファイル出力
- `Read` — 既存ファイルの確認
- `Bash(mkdir *)` — 出力先ディレクトリの作成

