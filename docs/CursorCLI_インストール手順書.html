01
Overview

Cursor CLI とは

Cursor CLI は、Cursor エディタの AI エージェントをターミナルから直接利用できるコマンドラインツールです。 自然言語の指示でコードの記述・レビュー・デバッグ・テストを行えます。GUI を持たないサーバー環境や、 CI/CD パイプライン（ヘッドレス実行）でも動作するのが特徴です。

  • 対話モード（インタラクティブ）／非対話モード（スクリプト・CI 向け）の両方に対応
  • コマンド名は cursor-agent（新しいエイリアスとして agent も利用可）
  • macOS / Linux / Windows（WSL・ネイティブ）をサポート
i 利用にあたって Cursor CLI の利用には Cursor アカウントが必要です。ログイン後、お使いの Cursor プランの利用枠に応じて エージェント機能が使えます。インストール作業を始める前にアカウントを用意しておくとスムーズです。

Requirements

動作環境・前提条件

今回のインストール先サーバーは、本作業環境（LXC コンテナ）と同等スペックを想定しています。 参考として現環境のスペックは以下のとおりです。Cursor CLI 自体は軽量で、特別な高スペックは不要です。

必要なもの

  • x86_64 または arm64 の Linux 環境（Ubuntu / Debian など）
  • インターネット接続（インストールスクリプトとバイナリのダウンロードに使用）
  • curl がインストールされていること
  • Cursor アカウント（ログイン認証用）
✓ curl が入っていない場合 Ubuntu / Debian では以下で導入できます。
bash
$ sudo apt update && sudo apt install -y curl
i root 権限は不要 インストールはユーザーのホームディレクトリ配下（~/.local/bin）に行われるため、 sudo / root 権限は不要です。一般ユーザーで実行してください。
03
Install

インストール

公式のインストールスクリプトを実行します。Linux（および macOS / WSL）では次の 1 行のみです。

Linux / macOS / WSL — bash
$ curl https://cursor.com/install -fsS | bash

スクリプトが自動でお使いの環境に合ったバイナリを取得し、~/.local/bin 配下に cursor-agent を配置します。完了すると PATH 設定に関する案内が表示されます。

! パイプ実行（curl | bash）について ダウンロードしたスクリプトをそのまま実行する形式です。内容を事前に確認したい場合は、 一度ファイルに保存してから中身を確認・実行することを推奨します。
bash — verify before run
# 1) スクリプトを保存
$ curl https://cursor.com/install -fsS -o cursor-install.sh
# 2) 中身を確認
$ less cursor-install.sh
# 3) 実行
$ bash cursor-install.sh
参考：Windows ネイティブ（PowerShell）の場合
PowerShell
irm 'https://cursor.com/install?win32=true' | iex

※ 今回の Linux サーバーへの導入では使用しません。

04
PATH

PATH の設定

インストール先の ~/.local/bin が PATH に含まれていないと、 cursor-agent コマンドが見つかりません。多くの場合インストールスクリプトが 自動で追記しますが、反映されない場合は以下を手動で実行してください。

Bash を使っている場合

~/.bashrc
$ echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
$ source ~/.bashrc

Zsh を使っている場合

~/.zshrc
$ echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
✓ 確認のコツ 使用中のシェルは echo $SHELL で確認できます。設定後、新しいターミナルを開き直すと確実です。
05
Verify

インストールの確認

バージョンが表示されればインストール成功です。

verify
$ cursor-agent --version
# 新しいエイリアスでも確認できます
$ agent --version
! 「command not found」と出る場合 PATH が通っていない可能性があります。手順 04（PATH の設定）を実施し、 ターミナルを開き直してから再度お試しください。
06
Authentication

認証（ログイン）

コマンドを使う前に Cursor アカウントでの認証が必要です。用途に応じて 2 つの方法があります。

方法 A — ブラウザでログイン（通常の対話利用向け）

login
$ cursor-agent login

実行するとブラウザが開き、認証を求められます。GUI の無いサーバーの場合は、 表示される URL を手元の PC のブラウザで開いて認証してください。認証状態は次のコマンドで確認できます。

status / logout
$ cursor-agent status   # 認証状態とアカウント情報を表示
$ cursor-agent logout   # ログアウト（認証情報をクリア）

方法 B — API キー（自動化・CI/CD・ヘッドレス向け）

Cursor のダッシュボード（Dashboard → Integrations → API Keys）でユーザー API キーを発行し、 環境変数 CURSOR_API_KEY に設定します。ブラウザ認証が使えないサーバーで便利です。

env — recommended
$ export CURSOR_API_KEY=your_api_key_here
$ cursor-agent "implement user authentication"
flag
$ cursor-agent --api-key your_api_key_here "implement user authentication"
! API キーの取り扱い API キーは秘密情報です。コマンド履歴やソースコードに直接書き込まず、環境変数や シークレット管理機能で扱ってください。
07
Usage

基本的な使い方

認証が済んだら、すぐに使い始められます。

examples
# 対話モードで起動
$ cursor-agent

# プロンプトを直接渡して実行
$ cursor-agent "find one bug and fix it"

# 非対話（print）モード — スクリプトや CI 向け
$ cursor-agent -p "find and fix performance issues"

# JSON で結果を取得（自動処理向け）
$ cursor-agent -p --output-format json "summarize this repo"

よく使うコマンド・フラグ

目的コマンド / フラグ
非対話（出力のみ）モード-p / --print
出力フォーマット指定--output-format json / text / stream-json
プランモード（実装前に方針設計）--plan / 対話中は /plan
Ask モード（変更せず調査のみ）--mode=ask / 対話中は /ask
独立した worktree で編集--worktree
会話を再開cursor-agent resume / --resume [thread id]
過去の会話一覧cursor-agent ls
✓ 対話モードの操作 矢印キーで過去メッセージを参照、Shift+Enter で複数行入力、 Ctrl+R で変更内容のレビュー、Ctrl+D で終了できます。 ターミナルコマンドの実行前には承認を求められます。
08
Maintenance

更新・アンインストール

アップデート

Cursor CLI はデフォルトで自動更新を試みます。手動で更新する場合は次のコマンドを使います。

update
$ cursor-agent update

アンインストール

インストール済みバイナリはホームディレクトリ配下にあります。削除するには実体を消し、 PATH 追記行を設定ファイルから取り除きます。

uninstall
# バイナリの場所を確認
$ which cursor-agent
# 例: ~/.local/bin 配下の関連ファイルを削除
$ rm -f ~/.local/bin/cursor-agent ~/.local/bin/agent
# 必要に応じて設定/キャッシュも削除（存在する場合）
$ rm -rf ~/.cursor ~/.config/cursor-agent
i 補足 ~/.bashrc / ~/.zshrc に追記した export PATH=... の行も、不要であれば手動で削除してください。

Troubleshooting

トラブルシューティング

症状対処
cursor-agent: command not found 手順 04 で PATH を設定し、ターミナルを開き直す
認証エラー / 未ログイン cursor-agent login を実行、または CURSOR_API_KEY の設定を確認
SSL 関連のエラー（開発環境） --insecure フラグを使用（本番環境では非推奨）
独自エンドポイントに接続したい --endpoint フラグでカスタム API エンドポイントを指定
curl が無い sudo apt install -y curl で導入
